
# 연산자

      연산자(operator) -> 하나 이상의 포현식을 대상으로 산술, 할당, 비교, 논리, 타입, 지수 연산(ooperation)등을 수행해
      
      하나의 값을 만든다.
      
      피연산자(opreand) : 연산의 대상
    
      ⭐ 피연산자 : '값'이라는 명사의 역할
      
      ⭐ 연산자 : '피연산자를 연산하여 새로운 값을 만든다'라는 동사의 역할
      
  
   ***
   
   ### 산술 연산자
   
      피연산자를 대상으로 수학적 계산을 수행해 새로운 숫자 값을 만든다.
      
      ❗ 산술이 불가능한 경우, NaN 반환
      
      
      ⭐ 피연산자의 개수에 따라 '이항 산술 연산자'와 '단항 산술 연산자'로 구분
      
            🟠 이항 산술 연산자 : 2개의 피연산자를 산술 연산하여 숫자 값을 만든다.
            
                                  피연산자의 값을 변경하는 부수 효과(side effect)가 없고, 언제나 새로운 값을 만든다.
                                  
            🟠 단항 산술 연산자 : 1개의 피연산자를 산술 연산하여 숫자 값을 만든다.
            
                                  +        -> 부수 효과 업음 (음수 -> 양수 반전 안함)
                                  
                                  -        -> 부수 효과 없음    ⭐ 양수 -> 음수, 음수 -> 양수로 반전한 값 반환
                                  
                                  +, -     -> 부수 효과 없음
                                  
                                  +=, --    -> 부수 효과 있음   ⭐ 피연산자의 값을 변경하는 암묵적 할당이 이뤄진다!
                                  
                                                   
               📌 전위 증가 / 감소 연산자 
               
                   ● x++ / x--    : 먼저 다른 연산을 수행한 후, 피연산자의 값을 증가/ 감소 
                    
                   ● ++x / --x    : 먼저 피연산자의 값을 증가/감소 시킨 후, 다른 연산 수행 
                             
                             
               ⭐⭐ 문자열은 숫자로 타입 변환할 수 없으므로 NaN 반환
               
                    -Hello';      // NaN
      
      ***
      
 ### 문자열 연결 연산자
 
        + 연산자는 피연산자 중 하나 이상이 문자열인 경우 문자열 연결 연산자로 동작.
        
            '1' + 2;      // '12'
            
        
        ⭐ null은 0으로 타입 변환
        
              1 + null;     // 1
              
        ⭐ undefined는 숫자로 타입 변환 x
        
              +undefined;       // NaN
              
              1 + undefined;    // NaN
                   
        ⭐⭐ 자바스크립트 엔진에 의해 암묵적 타입 변환(implicit coercion) (또는 타입 강제 변환. type corecion)이 일어나기도 한다.
        
                1 + true;   // 2
                
                +/- 단항 연산자
                
 ***
 
 ### 할당 연산자
      
          : 우항에 있는 피연산자의 평가 결과를 좌항에 있는 변수에 할당.
          
          -> 좌항의 변수에 값을 할당하는거니까, 변수 값이 변하는 부수 효과가 있다. 
          
          ❗ 할당문은 표현식인 문일까, 표현식이 아닌 문일까?
          
              -> 값으로 평가되는 표현식인 문으로서 할당된 값으로 평가된다.
              
 ***
 
### 비교 연산자

          : 좌항과 우항의 피연산자를 비교한 다음 그 결과를 불리언 값으로 변환.
          
          ⭐ if문이나 for 문과 같은 제어문의 조건식에서 주로 사용.
          
          
          🟠 동등 비교 연산자 -> 느슨한 비교    ⭐ 암묵적 타입 변환을 통해 타입을 일치시킨다. 같은 값인지 비교. 
          
                ==    : 동등 비교   
                !=    : 부동등 비교
          
          🟠 일치 비교 연산자 -> 엄격한 비교!   ⭐ 타입도 같고 값도 같은 경우. 암묵적 타입 변환을 하지 않고 값을 비교한다. 
                                                  -> 예측하기 쉬움. 
          
                ===   : 일치 비교
                !==   : 불일치 비교
                
            ⭐ 동등 비교와 일치 비교 모두 부수 효과는 없다.
            
            ⭐⭐ 동등 비교(==) 연산자는 예측하기 어려운 결과를 만들어내기 때문에,
            
                  사용하지 않는 편이 좋다. 대신 일치 비교(===) 연산자 사용!
                  
            
            🔥 일치 비교 연산자에서 주의할 것은 NaN이다.
            
                    NaN === NaN;    // false        ⭐ NaN은 자신과 일치하지 않는 유일한 값이다.
                    
                    -> 숫자가 NaN인지 조사하려면 빌트인 함수 isNaN을 사용한다.
                    
                    * 빌트인 함수란? 
                
            🔥 숫자 0도 주의!
            
                 자바스크립트에는 양의 0과 음의 0이 있는데 일치 비교/동등 비교 모두 결과는 true이다.
                 
                      0 === -0;   // true
                      0 == -0;    // true


            🟨 Object.is 메서드
        
                  ES6에서 도입되었고, 예측 가능한 정확한 비교 결과를 반환한다.
                  
                        -0 === +0;             // true
                        Object.is(-0, +0);     // false
                        
                        NaN === NaN;           // false
                        Object.is(NaN, NaN);   // true

   ### 삼항 조건 연산자
   
          : 두번째 피연산자 또는 세번째 피연산자로 평가되는 표현식
          
            첫번째 피연산자는, 불리언 타입의 값으로 평가될 '표현식'이다.
            
            
            ⭐ 조건식의 평가 결과가 불리언 값이 아니면 불리언 값으로 암묵적 타입 변환된다.
            
            ⭐ 삼항 조건 연산자 표현식은 조건문이다.
            
                      조건문 ? true일때의 값 : false일때의 값
            
            
            ❗ if .. else문과의 중요한 차이점
                      
                삼항 조건 연산자 표현식은 값처럼 사용할 수 있지만 if...else 문은 값처럼 사용할 수 없다!
                
                -> if...else문은 표현식이 아닌 문이다.
                
                ⭐ 삼항 조건 연산자 표현식은 값으로 평가할 수 있는 '표현식인 문'이기 때문에,
                    
                    값처럼 다른 표현식의 일부가 될 수 있어 매우 유용하다!
                   
                   
                🧐 그럼 어떤 경우에 각각 사용하는게 좋을까?
                
                        조건에 따라 어떤 값을 결정해야 한다면 -> 삼항 조건 연산자 표현식 사용
                        
                        조건에 따라 수행해야 할 문이 하나가 아니라 여러개라면  -> 가독성 측면에서 if...else문 사용
